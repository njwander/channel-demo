# 渠道管理系统 V2 页面开发文档

本文档基于 `docs/business-rules.md` (V2 业务规则) 及现有系统现状生成，旨在定义 V2 版本的页面结构、字段定义及交互逻辑。V1 版本页面与逻辑主要参考 `docs/渠道管理系统-PRD.md`。

## 1. 页面体系概览

系统主要包含以下核心模块页面：

| 模块         | 页面名称           | 路径 (Route)             | 说明                                   |
| :----------- | :----------------- | :----------------------- | :------------------------------------- |
| **工作台**   | 工作台 (Dashboard) | `/dashboard`             | 核心指标、待办事项、快捷入口           |
| **入驻申请** | 入驻申请列表       | `/settlement-list`       | 渠道入驻申请记录                       |
|              | 新建入驻申请       | `/settlement-new`        | 发起新的渠道入驻                       |
|              | 入驻申请详情       | `/settlement-detail/:id` | 审核、签约详情                         |
| **渠道管理** | 渠道列表           | `/channel-list`          | 正式合作渠道的管理                     |
|              | 渠道详情           | `/channel-detail/:id`    | 渠道全景视图（基础、报备、业绩、结算） |
|              | 渠道解约           | `/termination`           | 解约记录管理（入口主要在详情页）       |
| **客户报备** | 报备列表           | `/reporting-list`        | 客户报备记录与状态跟踪                 |
|              | 新建报备           | `/reporting-new`         | 录入报备信息                           |
|              | 报备详情           | `/reporting-detail/:id`  | 报备审查与进度详情                     |
| **业绩管理** | 月度结算           | `/monthly-settlement`    | 每月业绩结算单管理                     |
|              | 订单核对           | `/order-reconciliation`  | 订单明细查询与核对                     |

## 2. 页面功能详情

### 2.1 工作台 (Dashboard)

**主要变化 (V2)**:

- 待办事项需增加“报备审查”类任务。
- 漏斗模型需适配新的报备状态流转。

#### 2.1.1 核心区域

1.  **顶部欢迎与时间**: 展示用户昵称及当前日期。
2.  **待办事项 (To-Do List)**:
    - **报备审查**: "你有 X 条新的客户报备待审查" (跳转 `reporting-list?status=pending`)。
    - **入驻审核**: "【渠道名】入驻申请待审批" (跳转 `settlement-list?status=pending`)。
    - **即将到期**: "【渠道名】合作即将到期"。
    - **结算核对**: "X月结算单待确认"。
3.  **核心指标卡片**:
    - **渠道**: 负责渠道数、合作中、即将到期、本月新入驻。
    - **报备**: 报备总数、保护期内、已转化 (成交)、本月新增。
4.  **图表区**:
    - **渠道业绩排行**: 柱状图。
    - **报备转化漏斗**: 待处理 -> 保护期 -> 报备成功(已宣讲) -> 已转化(已签约)。

---

### 2.2 入驻申请 (Settlement)

**逻辑**: 负责处理渠道从“申请”到“签约”的过程。

#### 2.2.1 入驻申请列表 (`/settlement-list`)

- **筛选**: 渠道名称、审批状态 (待审批/审批中/驳回/通过/已归档)、负责人、申请时间。
- **列表字段**: 申请单号、渠道名称、影刀负责人、申请时间、当前状态。
- **操作**: 查看详情、修改 (仅驳回状态)。

#### 2.2.2 新建入驻申请 (`/settlement-new`)

采用**向导式 (Wizard)** 布局，分两步：

- **Step 1: 基础信息**
  - **智能解析**: 支持上传营业执照 OCR 识别。
  - **字段**: 公司全称 (查重)、纳税人识别号、省市区、行业、企业规模、地址。
  - **联系人**: 姓名、手机号、邮箱、职位。
- **Step 2: 分佣配置**
  - **分佣方案**:
    1.  **影刀标准分佣**: 默认阶梯规则。
    2.  **自定义阶梯**: 手动配置区间和比例。
    3.  **固定比例**: 单一比例。
    4.  **聚水潭专享**: 特殊规则。
  - **提交**: 生成入驻申请单。

#### 2.2.3 入驻申请详情 (`/settlement-detail/:id`)

- **状态条**: 待审批 -> 待签约 -> 已归档。
- **信息展示**: 基础信息、分佣配置、联系人。
- **操作区**:
  - **审批**: 用/拒绝 (仅审批人)。
  - **签约**: 发起电子签 或 上传线下合同 (审批通过后)。
  - **归档**: 签约完成后系统/人工归档，归档后自动生成**正式渠道**数据。

### 2.3 渠道管理 (Channel)

**主要变化 (V2)**:

- 渠道解约入口移至详情页。
- 合同信息合并至基础资料 Tab。
- 联系人展示优化为表单形式。

#### 2.3.1 渠道列表 (`/channel-list`)

- **筛选**: 渠道名称、状态 (合作中/即将到期/已解约)、分佣类型、负责人。
- **列表字段**: 渠道名称、状态、分佣类型、当前比例、累计业绩、成交客户数、负责人。
- **操作**: 查看详情。

#### 2.3.2 渠道详情 (`/channel-detail/:id`)

- **头部 (Header)**: 渠道名称、状态标签、负责人、核心指标 (本周期业绩、累计客户、保护中)。
- **Tabs 布局**:
  1.  **基础资料 (Basic Info)**:
      - 工商信息、地址。
      - **合同/分佣信息**: 签约周期、分佣规则、合同附件 (下载/预览)。
      - **联系人信息**: 姓名、电话、邮箱、职位 (单人表单样式)。
  2.  **报备管理 (Reporting)**:
      - 关联该渠道的客户报备列表 (复用报备列表组件，锁定渠道筛选)。
      - 快捷操作: [新建报备]。
  3.  **成交业绩 (Performance)**:
      - 关联的 OMS 订单列表 (日期、金额、回款、分佣)。
  4.  **结算历史 (Settlement)**:
      - 该渠道的历史月度结算单列表。
- **更多操作 (右上角)**:
  - **调整业绩周期**: 修改合作起止时间。
  - **移交负责人**: 变更影刀侧对接人。
  - **解约**: 发起解约流程 (填写原因、凭证)，状态变更为“已解约”。

### 2.4 客户报备 (Reporting) - **核心升级**

**依据**: `docs/business-rules.md` 3.2.2 章节。
**核心逻辑**: 报备不再是一步到位，而是生成“审查任务”。

#### 2.4.1 报备列表 (`/reporting-list`)

- **筛选**:
  - 客户名称、渠道名称。
  - **报备状态**:
    - **待处理 (Pending)**: 初始状态，等待自动/人工审查。
    - **保护期 (Protection)**: 审查通过，进入倒计时 (7/15天)。
    - **报备成功 (Success)**: 保护期内有效宣讲。
    - **报备失败 (Fail)**: 撞单、资料已存在、人工驳回。
    - **报备失效 (Invalid)**: 保护期过期未宣讲。
    - **报备流失 (Lost)**: 掉回公海。
    - **已转化 (Converted)**: 产生签约订单。
  - 负责人、报备时间。
- **列表字段**: 客户名称、渠道、报备人、状态 (带颜色标签)、保护截止日 (倒计时)、关联 CRM 客户、关联 CRM 销售。
- **操作**: 详情、处理 (待处理状态下)。

#### 2.4.2 新建报备 (`/reporting-new`)

- **数据源**: 通常由渠道接口人提供信息，影刀渠道负责人/销售代录入。
- **字段**:
  - **客户名称** (必填): 企业全称，触发企查查/CRM 初筛。
  - **关联渠道** (必填): 选择归属渠道。
  - **联系人信息**: 姓名、电话、职位。
  - **报备截图/凭证**: 图片上传。
  - **推荐销售** (选填): 文本输入，建议的跟进销售。
- **提交结果**: 创建一条状态为 **"待处理"** 的报备记录 (即审查任务)。

#### 2.4.3 报备详情/审查 (`/reporting-detail/:id`)

**审查任务流**:

1.  **信息展示**: 报备的基础信息、凭证。
2.  **自动化校验结果**:
    - 企查查校验: 是否存在。
    - System Check: 自动规则通过/不通过。
3.  **审查操作区 (Review Action)** - _针对“待处理”状态_:
    - **场景 A: 客户不存在**:
      - 操作: **[创建客户并关联]**。
      - 逻辑: 调用 CRM 接口创建客户 -> 关联渠道 -> 状态流转为 **"保护期"**。
    - **场景 B: 客户已存在 (公海/无跟进)**:
      - 操作: **[认领客户并关联]** (若是销售) 或 **[转交销售]** (若是渠道经理)。
      - 逻辑: 认领 -> 关联渠道 -> 状态流转为 **"保护期"**。
    - **场景 C: 客户已存在 (撞单/跟进中)**:
      - 操作: **[判定撞单/失败]**。
      - 逻辑: 录入原因 -> 状态流转为 **"报备失败"**。
4.  **保护期管理** - _针对“保护期”状态_:
    - 展示: 剩余天数 (倒计时)。
    - 校验: 是否有“宣讲”记录。
    - 操作: **[延长保护期]** (如有特批逻辑)、**[手动置为无效]**。
5.  **生命周期记录**: 记录状态流转的时间轴 (提交 -> 审查 -> 保护 -> 宣讲/成功 -> 签约 -> 转化)。

### 2.5 业绩与结算 (Performance)

#### 2.5.1 月度结算 (`/monthly-settlement`)

- **逻辑**: 每月自动生成，渠道确认，财务锁定，CEO 审批。
- **列表**: 结算单号、月份、渠道、业绩总额、分佣金额、状态 (待核对/待审批/已完成)。
- **详情**:
  - **业绩明细表**: 列出该月该渠道所有纳入计算的订单。
  - **调整记录**: 手动调账的记录。
  - **操作**: 确认核对、发起审批。

#### 2.5.2 订单核对 (`/order-reconciliation`)

- **逻辑**: 简化的订单查询视图。
- **功能**: 查询 OMS 订单，查看其归属渠道及计算的分佣金额。主要用于排查业绩疑问。
